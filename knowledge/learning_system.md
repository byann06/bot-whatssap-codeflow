# Learning System in Code Flow Community

## Filosofi Pembelajaran
Code Flow menggunakan sistem belajar bersama yang fokus pada:
- praktik langsung
- diskusi
- kolaborasi
- berkembang bersama
- learning by teaching

Code Flow percaya bahwa:
- semua orang bisa belajar teknologi
- error adalah bagian dari proses belajar
- skill berkembang secara bertahap
- tidak harus langsung jago untuk mulai belajar

## Learning by Teaching
Code Flow menggunakan konsep "learning by teaching".

Artinya:
anggota belajar sambil mengajar.

Karena tidak ada pemateri tetap, setiap anggota dapat menjadi pemateri secara bergantian.

Tujuan sistem ini:
- melatih hard skill
- melatih soft skill
- melatih public speaking
- meningkatkan rasa percaya diri
- membuat anggota lebih aktif

## Sistem Pemateri
Biasanya:
- 1 materi dibawakan oleh 2-3 anggota
- materi dibagikan sebelum pertemuan
- anggota dapat mempelajari materi terlebih dahulu
- pemateri boleh menggunakan AI atau browser sebagai alat bantu belajar

Pemateri bukan pusat utama pembelajaran.

Semua anggota dapat:
- berdiskusi
- membantu menjawab pertanyaan
- saling menjelaskan materi

## Format Pembelajaran
Format kegiatan biasanya:
- 30% teori
- 70% praktik dan diskusi

Code Flow lebih fokus pada praktik langsung dibanding teori panjang.

## Budaya Belajar
Budaya belajar di Code Flow:
- santai tapi tetap serius belajar
- tidak ada senioritas
- tidak merendahkan pemula
- tidak takut bertanya
- saling membantu
- saling berkembang

## Tentang AI dan Teknologi
Code Flow tidak menganggap AI sebagai hal negatif.

AI digunakan sebagai:
- alat bantu belajar
- alat bantu memahami materi
- alat bantu brainstorming
- alat bantu produktivitas

Namun anggota tetap didorong untuk:
- memahami konsep
- mencoba sendiri
- berdiskusi dengan anggota lain

## Tentang Error dan Kesalahan
Di Code Flow:
- error dianggap normal
- gagal dianggap bagian dari belajar
- anggota tidak perlu takut mencoba

## Tentang Project
Project dianggap sebagai salah satu cara belajar terbaik.

Anggota didorong untuk:
- membuat project kecil
- belajar teamwork
- belajar problem solving
- belajar workflow project nyata

## Tentang Diskusi
Diskusi adalah bagian penting komunitas.

Anggota didorong untuk:
- bertanya
- memberikan pendapat
- memberikan kritik dan saran
- membantu anggota lain memahami materi
